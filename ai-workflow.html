AI Workflow in My Project Folder

This is how I use a Claude Code agent inside my project folder to drive end-to-end changes across the R analysis and the LaTeX writeup, with collaborators picking changes up via Overleaf.

Project folder structure

  • Data/ — raw and cleaned datasets
  • Scripts/ — R analysis scripts; figures, tables, and text outputs land in LaTeX/Output/
  • LaTeX/ — paper source
  • Literature/ — reading material
  • Feedback/ — coauthor and supervisor feedback, plus meeting notes

The LaTeX/ folder is its own git repo that syncs to Overleaf; the rest of the project folder syncs to GitHub.

Example: adding an appendix figure

I open a Claude Code agent inside the project folder and give it a prompt like this.

Add an appendix figure that compares the estimated treatment effects across genders. Cite it as a robustness analysis in the main text where I discuss heterogeneity, and include the takeaway in that paragraph. Follow the same method I used for the age breakdown — treat gender as a new heterogeneous variable. Before making any changes, ask questions to resolve any uncertainties.

What the agent does

  1. Asks clarifying questions first, then updates the relevant R script under Scripts/ to add gender as the heterogeneous variable, mirroring the age breakdown.
  2. Runs the R script; the new figure is saved into LaTeX/Output/Figures/.
  3. Updates the LaTeX source: adds a new appendix section "Robustness: Gender Effects" that includes the figure, and edits the main-text heterogeneity paragraph to add the takeaway with a correct cross-reference to the new appendix.
  4. I review the diff across R, LaTeX, and Output — tweak or send follow-up instructions if needed.
  5. I push the LaTeX/ folder to Overleaf with git; coauthors see the new appendix and updated text.
