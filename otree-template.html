oTree Template

GitHub

An oTree starter scaffold for running lab experiments. Plug your instructions and your game into the main app — the welcome, consent, comprehension quiz, demographics, and payment flow are already wired in across the before, intro, and outro apps.

App timeline

A session moves through the four apps in order:

  • before — landing page, welcome message, and informed consent.
  • intro — task instructions followed by a comprehension quiz.
  • main — the experimental task itself, including any treatment-specific pages.
  • outro — demographic questionnaire, debrief, and payment information.

How to use it

Clone the repository to get a working copy:

git clone https://github.com/juliantait/Template_oTree.git

Then edit the files relevant to your experiment. The intended entry points are:

  • Instructions — edit intro/instructions_text.html. Each <div class="instruction-block"> renders as one participant page.
  • Comprehension quiz — define quiz items (prompt, choices, correct answer) in intro/quiz_items.py.
  • Treatment assignment — set up your conditions in before/treatment_assignment.py. Treatments are assigned at session creation via creating_session in the before app.
  • Game — build your economic game in the main app. Add any HTML pages for the game to this app, and configure the app settings (rounds, decisions, payoffs) in main/__init__.py.
  • Payment logic — decide how participants are paid in outro/payment_rule.py. The function there controls which rounds and payoffs count toward the final payment.

Previewing instructions

Iterating on instruction copy without spinning up the full oTree server is awkward, so the template ships with a preview generator. Running previews/generate_instructions_preview.py produces three self-contained files in previews/ — CSS and JavaScript are inlined, so each one can be emailed or attached without external dependencies.

  • instructions_preview_long.html — every instruction block stacked vertically in its normal card chrome, with a treatment dropdown at the top.
  • instructions_preview_interactive.html — a single page that mimics the live participant experience, with Back/Next navigation, floating treatment-switcher buttons in the top-right, and a setup screen for picking the starting treatment.
  • instructions_preview.pdf — the long view rendered to PDF for printing or sharing with coauthors who prefer paper.

Variables and {% if treatment %} conditional blocks render dynamically in the HTML versions; the PDF preserves variable names as literal placeholders. The script opens a tkinter popup for entering treatment × variable values, and falls back to a JSON config file for headless use.

Shared visual scaffold

All apps inherit a shared HTML/CSS layer that gives sessions a consistent look without per-app styling. The reference layout lives at _static/global/html/template.html and demonstrates the standard card layout used across pages — header, body content, and a navigation footer. Extending or restyling the experiment is a matter of editing this one template and the accompanying stylesheet.

Making oTree data usable

An R script that reshapes oTree's wide session CSVs into long format (one row per participant × app × round), pre-configured for the variable names used in this template. It is one helper from the Empirical Analysis Template — a self-contained R + LaTeX pipeline for a single-study empirical paper.
