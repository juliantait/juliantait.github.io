Empirical Analysis Template

GitHub

A self-contained R + LaTeX template for a single-study empirical paper. Provides the analysis-pipeline scripts, data-source adapters, paper template, and figure/table style conventions — set up your Data/, run main.R, write up in LaTeX/. Use it manually; no automation required.

Structure

  • Data/ — raw and cleaned data folders
  • Helper/ — pluggable data-source adapters (otree.R, csv.R)
  • Scripts/ — config_init, config_cleaning (orchestrator), descriptives, hypotheses, balance_table, robustness, sample_restrictions, exploratory
  • LaTeX/ — paper source. Analysis outputs land in LaTeX/Output/{Figures,Tables,Text}/, so the LaTeX folder is self-contained.
  • Literature/ — drop your reading material here
  • Flow/ — codebook, research_log, timeline, todo, personality, plus Flow/Tools/ (skill references for graphs and tables) — designed as both a human style guide and a drop-in point for integrating with Claude or other LLM agents.

Getting started

Clone the repository to get a working copy:

git clone https://github.com/juliantait/Template_Analysis.git
LaTeX — folder layout and Overleaf sync

The LaTeX/ folder holds the paper source files. R analysis writes its figures, tables, and inline numbers into LaTeX/Output/{Figures,Tables,Text}/, and the .tex files reference them via relative paths like Output/Figures/foo.png. The folder compiles standalone wherever it ends up — locally, on Overleaf, or in CI.

Sync to Overleaf

  1. Create a new empty project in Overleaf (web UI).
  2. In the project sidebar, open Integrations → Git and copy the project's git URL — e.g. https://git.overleaf.com/<project-id>.
  3. Inside the LaTeX/ folder, init a git repo if needed: cd LaTeX && git init. The LaTeX folder becomes its own working tree, independent from the outer repo.
  4. Add Overleaf as a remote and push: git remote add overleaf <overleaf-git-url> && git push -u overleaf main:master. Overleaf has historically used master as the default branch; if main:master is rejected, run git ls-remote overleaf to see the actual branch name.
  5. Subsequent edits: git push overleaf main:master from inside LaTeX/. Collaborator changes from Overleaf come back via git pull overleaf master.

Overleaf's git integration is a premium feature — available on paid individual or group subscriptions and to Overleaf Commons participants. Free-plan projects will not show a Git option in Integrations.

Full Overleaf-side guide: docs.overleaf.com — Git integration.
