Empirical Analysis Template

GitHub

A self-contained R + LaTeX template for a single-study empirical paper. Provides the analysis-pipeline scripts, data-source adapters, paper template, and figure/table style conventions — set up your Data/, run main.R, write up in LaTeX/. Use it manually; no automation required.

Structure

  • Scripts/ — config_init, config_cleaning (orchestrator), descriptives, hypotheses, balance_table, robustness, sample_restrictions, exploratory
  • Helper/ — pluggable data-source adapters (otree.R, csv.R)
  • LaTeX/ — paper source. Analysis outputs land in LaTeX/Output/{Figures,Tables,Text}/, so the LaTeX folder is self-contained.
  • Data/ — raw and cleaned data folders
  • Flow/ — codebook, research_log, timeline, todo, personality (where you record project aim)
  • Tools/ — JEBO-style conventions for publication-ready graphs and tables (skill_graphs.md, skill_tables.md)
  • Literature/ — drop your reading material here

Getting started

Clone the repository to get a working copy:

git clone https://github.com/juliantait/Template_Analysis.git

Outputs and Overleaf sync

Analysis scripts write all figures, tables, and inline numbers to LaTeX/Output/, split into Figures/, Tables/, and Text/. The .tex files reference these via relative paths, so the LaTeX folder compiles wherever you put it (local, Overleaf, a CI job).

Need outputs in a second place at the same time — e.g. an Overleaf clone? Add the path to SYNC_DESTINATIONS in Scripts/config_init.R and every save call writes to both.

Sync to Overleaf

The LaTeX/ folder is designed to be the working tree of an Overleaf project. The flow is local-first: create a new empty Overleaf project, then push the local LaTeX/ contents up to it.

  1. Create a new empty project in Overleaf (web UI).
  2. In the project sidebar, open Integrations → Git and copy the project's git URL. URLs look like https://git.overleaf.com/<project-id>.
  3. In a terminal, from inside the local LaTeX/ folder, initialise a git repo if it is not one already: 
    cd LaTeX && git init
    The LaTeX folder is its own working tree, independent from the outer Template_Analysis repo.
  4. Add Overleaf as a remote and push: 
    git remote add overleaf <overleaf-git-url> && git push -u overleaf main:master
    Overleaf has historically used master as the default branch; if the push is rejected, git ls-remote overleaf reveals the actual branch name.
  5. Subsequent edits: git push overleaf main:master from inside LaTeX/. To pull collaborator edits back from Overleaf, git pull overleaf master.

Note: Overleaf's git integration is a premium feature — available on paid individual or group subscriptions and to Overleaf Commons participants. Free-plan projects will not show a Git option in Integrations.
